# Root Docs

This folder is the agent-friendly knowledge base for the root repo.

## Core Topics

- [Architecture](../ARCHITECTURE.md): repo shape, module catalog, preset fixtures, and ownership boundaries
- [Reliability](RELIABILITY.md): root repo commands, workflow coverage, fixture freshness, and release verification
- [Plans](PLANS.md): planning model and execution-plan conventions
- [Quality Score](QUALITY_SCORE.md): review rubric for modules, fixtures, docs, and automation changes
- [Security](SECURITY.md): supply-chain and secret-scanning expectations

## Suggested Reading Order

1. Read `ARCHITECTURE.md` to understand the repo layout.
2. Read `docs/RELIABILITY.md` before changing packaging, workflows, repo commands, module validation, or scaffold verification.
3. Read `docs/PLANS.md` before creating or editing execution plans.
4. Read `docs/QUALITY_SCORE.md` when reviewing doc, module, fixture, or automation changes.
5. Read `docs/SECURITY.md` when changing supply-chain checks or publication posture.
