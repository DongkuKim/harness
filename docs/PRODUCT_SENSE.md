# Product Sense

This repo is optimized for agents and developers who want a small, opinionated module catalog for Next.js, Python, and Rust projects.

The default experience should communicate:

- the supported modules are intentionally small and explicit
- module selection is based on visible kind, runtime, repeatability, and dependency metadata
- scaffold composition reuses committed module sources deterministically
- release and CI checks are part of the product promise, not just internal maintenance

Detailed specs live in `product-specs/`.
