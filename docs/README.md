# Root Docs

This folder is the agent-friendly knowledge base for the root repo.

## Core Topics

- [Architecture](../ARCHITECTURE.md): repo shape, template catalog, and ownership boundaries
- [Release](RELEASE.md): npm packaging and scaffold smoke checks
- [CI](CI.md): GitHub workflow coverage and lane structure
- [Expectations](EXPECTATIONS.md): planning, reliability, and quality rules

## Suggested Reading Order

1. Read `ARCHITECTURE.md` to understand the repo layout.
2. Read `docs/RELEASE.md` before changing packaging or the CLI.
3. Read `docs/CI.md` before changing workflow behavior or template validation.
4. Read `docs/EXPECTATIONS.md` before making broad changes to project standards.
